# Checklist de Entrega al Cliente

Todo lo que tiene que estar listo el día que se cierra el proyecto.

---

## Código y Repositorio
- [ ] Repo en GitHub limpio (sin archivos .env, sin console.log, sin código comentado)
- [ ] README con instrucciones claras para correr el proyecto localmente
- [ ] Variables de entorno documentadas en `.env.example`
- [ ] Último commit con mensaje descriptivo, no "fix" ni "wip"

## Accesos a entregar
- [ ] Credenciales del panel de administración (usuario + contraseña temporal)
- [ ] Acceso a Supabase (invitar email del cliente como Owner)
- [ ] Acceso al hosting (DigitalOcean / Vercel — agregar como colaborador)
- [ ] Acceso al dominio si lo compré yo (transferir o agregar como manager)
- [ ] Credenciales de Mercado Pago si las configuré yo
- [ ] Google Search Console verificado y acceso entregado

## Documentación
- [ ] Manual básico de uso del CMS / panel admin (capturas de pantalla o video Loom)
- [ ] Lista de tecnologías usadas con versiones
- [ ] Cómo hacer deploy de cambios (si el cliente va a tocar código)
- [ ] Contactos de soporte: hosting, dominio, base de datos

## Producción
- [ ] Dominio final apuntado (no subdominio de prueba)
- [ ] SSL activo y renovación automática
- [ ] Emails transaccionales funcionando (confirmación de formularios, etc.)
- [ ] Webhooks de pago activos y testeados con transacción real

## Soporte post-entrega
- [ ] Acordar 30 días de soporte incluido
- [ ] Canal de comunicación definido (WhatsApp / email)
- [ ] Límite de soporte claro: bugs del proyecto entregado, no nuevas features
- [ ] Cotizar mantenimiento mensual si el cliente lo quiere (ARS 80.000–150.000/mes)
